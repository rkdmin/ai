## 전체 단계 한눈에 보기

```
Phase 1 │ 얼굴 인식 정확도 + RAG 품질
Phase 2 │ 백엔드 분리 + 보안
Phase 3 │ 인증 + 유저 시스템
Phase 4 │ UI/UX 완성도
Phase 5 │ 수익화 시스템
Phase 6 │ 모바일 앱 (Android / iOS)
```

| Phase | 핵심 목표 | 세부 계획 파일 | 완료 기준 |
|-------|----------|--------------|---------|
| 1 | 얼굴 인식 정확도 개선 + RAG 데이터 보강 | `phase1-quality.md` | 사용자 자가평가 정확도 80% 이상 |
| 2 | 백엔드 API 서버 분리 | `phase2-backend.md` | API 키 클라이언트 노출 없음 |
| 3 | 로그인 + 히스토리 저장 | `phase3-auth.md` | 카카오/구글 로그인 + 내 기록 조회 |
| 4 | UI/UX 폴리싱 | `phase4-ux.md` | 디자인 시스템 확립 + 온보딩 완성 |
| 5 | 수익화 (쿠팡파트너스 + 광고) | `phase5-monetization.md` | 첫 수익 발생 |
| 6 | 모바일 앱 출시 | `phase6-mobile.md` | 앱스토어 + 플레이스토어 출시 |

---

## 우선순위 원칙

1. **신뢰도 먼저** — RAG 기반 추천이 앱의 핵심 가치. 정확하지 않으면 나머지 다 무의미.
2. **보안 필수** — API 키 클라이언트 노출은 과금 폭탄 리스크. Phase 2는 Phase 3 전에 반드시 완료.
3. **수익화는 UX를 해치지 않게** — 광고/제휴가 추천 신뢰도를 낮춰선 안 됨.
4. **모바일은 웹 완성 후** — 웹 MVP가 검증된 후 모바일 전환.
