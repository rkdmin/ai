> 선행 조건: Phase 1 (정확도) 완료 권장

## 4-1. 하단 탭바 (Bottom Navigation)

```
┌──────────────────────────────┐
│                              │
│         (화면 내용)           │
│                              │
├──────────────────────────────┤
│   🏠 홈   │ ✨ 트렌드 │ 📋 히스토리 │
└──────────────────────────────┘
```

- 탭 3개: 홈(분석) / 트렌드 / 히스토리
- 현재 탭 아이콘 + 텍스트 활성 색상 (`--rose`)
- 홈 탭 내 분석 플로우 진행 중에는 탭바 숨김 (분석 집중)
  - `analyzing`, `generatingCards` 단계에서 탭바 hidden
- iOS safe area 하단 여백 처리 필수

## 4-3. 모바일 최적화

### 기준 해상도
- **메인 타깃**: iPhone 14 (390px), Galaxy S23 (360px)
- **서브 타깃**: iPad (768px), 소형폰 (320px)

### 해야 할 것

- [ ] 모든 화면 375px 기준으로 레이아웃 재검토
- [ ] 터치 타겟 최소 44×44px 확보
- [ ] 스크롤 오버스크롤 바운스 효과 (`overscroll-behavior`)
- [ ] iOS Safari 하단 safe area 처리 (`padding-bottom: env(safe-area-inset-bottom)`)
- [ ] 이미지 업로드: 카메라 직접 촬영 버튼 추가 (`accept="image/*;capture=camera"`)
- [ ] 키보드 올라올 때 레이아웃 깨지지 않도록 처리
- [ ] 핀치 줌 방지 (`viewport: user-scalable=no` — 필요한 화면만)

## 4-4. 온보딩 플로우

> 첫 방문 사용자에게 앱 사용법 안내

### 온보딩 화면 구성 (스와이프 슬라이드 3장)

```
슬라이드 1:
┌──────────────────────────────┐
│   📸                         │
│   사진 한 장으로              │
│   나의 얼굴형을 분석해요      │
│                              │
│   정면 + 측면 사진으로        │
│   더 정확한 결과를 받을 수 있어요 │
└──────────────────────────────┘

슬라이드 2:
┌──────────────────────────────┐
│   💄                         │
│   AI가 나에게 딱 맞는         │
│   헤어·메이크업을 추천해요    │
│                              │
│   검증된 뷰티 데이터 기반으로  │
│   신뢰도 높은 추천             │
└──────────────────────────────┘

슬라이드 3:
┌──────────────────────────────┐
│   ✨                         │
│   추천 스타일을               │
│   내 얼굴에 바로 적용해봐요   │
│                              │
│   AI 합성 사진으로            │
│   미리 확인하세요             │
└──────────────────────────────┘

하단: [건너뛰기]  ○○● [시작하기]
```

### 구현 방법
- `localStorage`에 온보딩 완료 여부 저장
- 이후 방문 시 스킵

## 4-5. 촬영 가이드 개선

### 현재 문제
- 가이드 이미지만 보여줌 → 사용자가 잘못 찍는 경우 많음

### 해야 할 것

- [ ] 가이드 이미지에 체크리스트 추가
  ```
  ✓ 정면을 바라보고 있나요?
  ✓ 조명이 충분한가요?
  ✓ 머리카락이 얼굴을 가리지 않나요?
  ✓ 안경을 벗었나요?
  ✓ 표정이 자연스럽게 이완되어 있나요?
  ```
- [ ] 나쁜 예시 / 좋은 예시 비교 이미지 추가
- [ ] 업로드 전 "사진이 이 조건을 충족하나요?" 체크 UI

## 4-6. 로딩 화면 개선

### 해야 할 것

- [ ] 분석 로딩 (15~20초) — 지루하지 않게 개선
  ```
  현재: 바 애니메이션
  개선: 단계별 진행 표시 + 메시지 순환
  - "얼굴 윤곽을 파악하고 있어요..." (0~5초)
  - "이목구비 특징을 분석하고 있어요..." (5~10초)
  - "나에게 맞는 스타일을 찾고 있어요..." (10~15초)
  - "거의 다 됐어요!" (15초~)
  ```
- [ ] 카드 생성 로딩 (10~15초)
  ```
  "추천 카드를 만들고 있어요..."
  진행 퍼센트 표시 (가상)
  ```
- [ ] 에러 화면 개선
  ```
  현재: 토스트 메시지만
  개선: 에러 상황별 안내 + 재시도 버튼
  - 네트워크 오류: "인터넷 연결을 확인해주세요"
  - 사진 품질 문제: 구체적 개선 방법 안내
  - 서버 오류: "잠시 후 다시 시도해주세요"
  ```

## 4-7. 카드 UI 개선

### 카드 목록 화면

- [ ] 카드 미리보기에 헤어/메이크업 핵심 키워드 더 명확하게
- [ ] 추천 순위 뱃지 디자인 개선 (Best Pick → 더 눈에 띄게)
- [ ] Avoid 카드는 시각적으로 구분 (흑백 처리 또는 빨간 테두리)

### 카드 상세 화면

- [ ] 섹션별 접기/펼치기 (헤어, 메이크업 항목이 많으면 스크롤 길어짐)
- [ ] 공유 기능 추가 (카드 이미지로 캡처하여 공유)
  ```
  [인스타 스토리 공유] [이미지 저장]
  ```
- [ ] 메이크업 색상 추천 시 컬러 칩 표시
  ```
  립: [●] 코랄 레드  ← 실제 색상 원
  블러셔: [●] 피치
  ```
