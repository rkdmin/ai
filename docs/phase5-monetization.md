# Phase 5 — 수익화 시스템 (v1.1)

> ⚠️ **이 단계는 v1.1 — 즉 Android 1차 출시(v1.0) 이후에 진행한다.**
> v1.0 출시는 광고/카드 잠금 없이 진행하며, 수익화는 운영 지표 확인 후 도입한다.
> 단, 메이크업 카드의 쿠팡파트너스 정적 링크는 v1.0부터 노출한다 (별도 SDK 없음).
>
> 목표: 운영비를 회수하고, 추천 경험을 해치지 않는 구조로 수익화를 붙인다.
> 전략: 광고 + 쿠팡 파트너스 (광고는 v1.1, 쿠팡 링크는 v1.0부터)

## 5-1. 카드 잠금 구조 (보상형 광고) — v1.1 도입

```
v1.0 (Android 첫 출시):
  4장 모두 무료 공개 (광고/잠금 없음)

v1.1 (광고 도입 후):
  Rank 3 (3rd)  → 무료 공개
  Rank 1 (Best) → 잠금 + 보상형 광고 시청 후 해제
  Rank 2 (2nd)  → 잠금 + 보상형 광고 시청 후 해제
  Avoid 카드    → 무료 공개
```

### UX 흐름

1. Rank 1·2 카드는 블러 처리 + 자물쇠 아이콘 표시
2. 탭 시 `"광고를 보면 이 카드를 확인할 수 있어요"` 모달 노출
3. 광고 시청 완료 시 해당 카드만 해제
4. 세션 종료 후 다시 잠금

### 앱 구현 기준

- 패키지: `@capacitor-community/admob`
- **v1.1 업데이트에서 도입** (v1.0 출시 시점에는 모든 카드 무료)
- 광고 위치는 카드 추천 영역 바깥쪽에 두어 추천 신뢰도를 해치지 않게 배치

## 5-2. 쿠팡 파트너스 — 상품 큐레이션

### 노출 방식

```
기존: 메이크업 카드에도 AI 사진 적용 시도

변경:
메이크업 카드 상세 하단 추천 제품 2~4개 (퍼스널컬러 + 메이크업 파트 기반)
예) 봄웜 코랄 립틴트 / 피치 블러셔 / 누드 코랄 립스틱
```

### 구현 방향

- 초기 범위는 메이크업 카드 상세 하단 고정 영역
- 퍼스널컬러 + 메이크업 파트 키워드로 추천 제품 블록 구성
- 실제 `coupangPartnersUrl`은 운영 매핑 데이터 또는 서버 링크 빌더에서 생성
- AI는 링크를 직접 생성하지 않음
- 추후 필요 시 쿠팡 파트너스 API로 전환

### 표시 규칙

- 법적 고지 문구 필수:
  `"쿠팡 파트너스 활동의 일환으로, 이에 따른 일정액의 수수료를 제공받습니다."`
- 추천 카드 본문 내부가 아니라 상세 하단 별도 블록에 노출
- 메이크업 카드에서는 `"AI 사진 생성"` CTA를 노출하지 않음

## 5-3. 광고 시스템

### 웹

- 랜딩/웹 버전에는 Google AdSense 적용 가능
- 추천 핵심 영역 안쪽에는 광고를 두지 않는다

권장 위치:
- 카드 목록 하단
- 히스토리 목록 중간
- 트렌드 피드 카드 사이 (트렌드 탭 도입 후)

### 앱 (Capacitor)

| 광고 타입 | 위치 | 도입 시점 | 비고 |
|----------|------|---------|------|
| 배너 광고 | 카드 목록 하단 고정 | v1.1 | — |
| 보상형 광고 | 잠긴 카드 해제 | v1.1 | 핵심 수익화 |
| 전면 광고 | 분석 완료 후 결과 전환 직전 | v1.1 이후 검토 | 빈도 제한 필요, 이탈 리스크 |

- 과도한 전면 광고로 첫 사용자 이탈을 만들지 않음

## 5-4. 비용 높은 기능 운영 원칙

헤어/종합 카드의 사진 생성은 수익화 게이트가 아니라 사용량 제한으로 관리한다.

| 기능 | 운영 원칙 |
|------|----------|
| 얼굴 분석 | 게스트/로그인 상태에 따라 일일 제한 차등 적용 |
| 카드 생성 | 게스트/로그인 상태에 따라 일일 제한 차등 적용 |
| 헤어/종합 사진 생성 | "분석 1건당 카드 종류별 1회" + "일일 누적 상한" 두 단계로 관리 (게스트는 0회) |
| 히스토리 보관 | 최근 5회 기준으로 운영 |

### 사진 생성 사용량 정책 의미

- **분석 1건당 카드 종류별 1회**: 같은 분석에서 헤어 카드는 헤어 사진 1장, 종합 카드는 종합 사진 1장까지. 사용자가 자기 사진을 적용해보는 핵심 욕구를 막지 않는다.
- **일일 누적 상한**: 동일 유저가 하루에 분석을 여러 번 새로 하는 케이스 대비. v1.0 기본값 5회/일 (이상 시 안내 UI).
- 두 가드 모두 백엔드에서 검사하며, 프론트는 캐시 가능한 응답을 우선 보여준다.

### 헤어/종합 사진 생성 흐름

```
1. 카드 상세 → "AI 사진 생성" 버튼
2. 게스트면 로그인 CTA
3. 로그인 사용자면 /api/photo/generate 호출
   - 같은 (analysisId, cardType) 재요청 시 generated_photos UNIQUE → 캐시 응답 (사용자에겐 무한 재시도 가능)
   - 일일 누적 상한 초과 시에만 안내 UI 노출
4. 생성 성공 시 전후 비교 UI 노출
```

메이크업 카드 상세는 위 흐름 대신 추천 제품 블록과 쿠팡파트너스 링크를 노출한다.
