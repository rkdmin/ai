# Phase 5 — 수익화 시스템

> 목표: 운영비를 회수하고, 추천 경험을 해치지 않는 구조로 수익화를 붙인다.
> 전략: 광고 + 쿠팡 파트너스

---

## 수익 구조 설계

### API 비용 추산 (기준: 월 1,000명 사용자)

| 항목 | 사용 시점 | 예상 비용/회 | 월 추산 |
|------|----------|------------|--------|
| Gemini 분석 + 카드 생성 | 분석 1회 + 카드 생성 | ~$0.03 | $30/월 |
| Gemini 사진 생성 | 헤어/종합 카드 선택 시 | ~$0.01 | $5/월 |
| 합계 | | | **~$35/월** |

> 비용은 사용자 증가에 따라 가파르게 올라가므로 Phase 2의 백엔드 보호와 Rate Limiting이 전제다.

### 수익 모델

| 수익원 | 예상 수익 | 구현 우선도 |
|--------|---------|-----------|
| AdMob 보상형 광고 | 월 $100~400 | 높음 |
| AdMob 배너 광고 | 월 $30~100 | 높음 |
| 쿠팡 파트너스 TOP 3 | 월 $50~200 | 중간 |

---

## 5-1. 카드 잠금 구조 (보상형 광고)

```
카드 목록:
  Rank 3 (3rd)  → 무료 공개
  Rank 1 (Best) → 잠금
  Rank 2 (2nd)  → 잠금
  Avoid 카드    → 무료 공개
```

### UX 흐름

1. Rank 1·2 카드는 블러 처리 + 자물쇠 아이콘 표시
2. 탭 시 `"광고를 보면 이 카드를 확인할 수 있어요"` 모달 노출
3. 광고 시청 완료 시 해당 카드만 해제
4. 세션 종료 후 다시 잠금

### 앱 구현 기준

- 패키지: `@capacitor-community/admob`
- Android MVP에서 먼저 적용
- 광고 위치는 카드 추천 영역 바깥쪽에 두어 추천 신뢰도를 해치지 않게 배치

---

## 5-2. 쿠팡 파트너스 — 상품 큐레이션

### 노출 방식

```
기존: 메이크업 카드에도 AI 사진 적용 시도

변경:
메이크업 카드 상세 하단 추천 제품 TOP 3
1위 코랄 레드 립틴트
2위 피치 립글로스
3위 누드 코랄 립스틱
```

### 구현 방향

- 초기 범위는 메이크업 카드 상세 하단 고정 영역
- 퍼스널컬러 + 메이크업 파트 키워드로 추천 제품 블록 구성
- 실제 `coupangPartnersUrl`은 운영 매핑 데이터 또는 서버 링크 빌더에서 생성
- AI는 링크를 직접 생성하지 않음
- 추후 필요 시 쿠팡 파트너스 API로 전환

### 표시 규칙

- 법적 고지 문구 필수:
  `"쿠팡 파트너스 활동의 일환으로, 이에 따른 일정액의 수수료를 제공받습니다."`
- 추천 카드 본문 내부가 아니라 상세 하단 별도 블록에 노출
- 메이크업 카드에서는 `"AI 사진 생성"` CTA를 노출하지 않음

## 5-3. 광고 시스템

### 웹

- 랜딩/웹 버전에는 Google AdSense 적용 가능
- 추천 핵심 영역 안쪽에는 광고를 두지 않는다

권장 위치:
- 카드 목록 하단
- 히스토리 목록 중간
- 트렌드 피드 카드 사이

### 앱 (Capacitor)

| 광고 타입 | 위치 | 비고 |
|----------|------|------|
| 배너 광고 | 카드 목록 하단 고정 | MVP 포함 |
| 전면 광고 | 분석 완료 후 결과 전환 직전 | 빈도 제한 필요 |
| 보상형 광고 | 잠긴 카드 해제 | 핵심 수익화 |

### 운영 원칙

- 분석 중 로딩 화면에는 광고 금지
- 추천 이유 텍스트와 광고를 혼합하지 않음
- 과도한 전면 광고로 첫 사용자 이탈을 만들지 않음

---

## 5-4. 비용 높은 기능 운영 원칙

헤어/종합 카드의 사진 생성은 수익화 게이트가 아니라 사용량 제한으로 관리한다.

| 기능 | 운영 원칙 |
|------|----------|
| 얼굴 분석 | 게스트/로그인 상태에 따라 일일 제한 차등 적용 |
| 카드 생성 | 게스트/로그인 상태에 따라 일일 제한 차등 적용 |
| 헤어/종합 사진 생성 | 로그인 사용자 대상 일일 제한 적용 |
| 히스토리 보관 | 최근 5회 기준으로 운영 |

### 헤어/종합 사진 생성 흐름

```
1. 카드 상세 → "AI 사진 생성" 버튼
2. 게스트면 로그인 CTA
3. 로그인 사용자면 /api/photo/generate 호출
4. 일일 제한 초과 시 안내 UI 노출
5. 생성 성공 시 전후 비교 UI 노출
```

메이크업 카드 상세는 위 흐름 대신 추천 제품 블록과 쿠팡파트너스 링크를 노출한다.
